\subsection{Detailed Description}
This class inherits from \hyperlink{classmts_task}{mts\+Task} and provides the base for implementing continuous loops, where the user-\/supplied Run method is repeatedly invoked. It also introduces the possibility to create a new thread or to use an existing thread for the task.

If new\+Thread is false, the task will not create a new thread, but will rather \char`\"{}capture\char`\"{} an existing thread. This is done as follows\+:

1) Call \hyperlink{classmts_task_continuous_a28e79bc5f1a3eb9b7ef01910dd2b55e8}{mts\+Task\+Continuous\+::\+Create} from the thread to be captured. The class will get the thread id for the current thread and use it to initialize all interfaces in \hyperlink{classmts_task_a6816f8dfc6124376896318ebcdd4832b}{mts\+Task\+::\+Startup\+Internal}. Once this is done, Create returns to the calling thread.

2) Call \hyperlink{classmts_task_continuous_a935299c6e0bb33bb524fee95610a40ee}{mts\+Task\+Continuous\+::\+Start} from the thread to be captured. This function captures the thread, and uses it to call the \hyperlink{classmts_task_a20e18c9479cb650792456c32ee689834}{mts\+Task\+::\+Run} method. It does not return until the task is terminated.

Other tasks can use \hyperlink{classmts_component_a7264642eb5c56729d1f80263e0e1152b}{mts\+Task\+::\+Suspend} and \hyperlink{classmts_component_abfb4a63739ec45a9cdb91e3363db7c5c}{mts\+Task\+::\+Start} to suspend and resume this task, or \hyperlink{classmts_task_a67a15d7fffef28708b813e2ea624e4f8}{mts\+Task\+::\+Kill} to terminate it. In other words, once started the task should behave the same regardless of whether or not a new thread was created.

Note that \hyperlink{classmts_task_from_callback}{mts\+Task\+From\+Callback} provides another mechanism for using an existing thread. In that case, the task does not capture the thread, but just registers itself as a callback.
