\subsection{Detailed Description}
Declaration of \hyperlink{classosa_socket}{osa\+Socket}

This is a cross-\/platform socket library with basic support for U\+D\+P (datagram) and T\+C\+P (stream) sockets. The B\+S\+D socket A\+P\+I is used on Unix-\/like systems, while the Winsock2 A\+P\+I is used on Windows. 

For U\+D\+P, both a server and client can be defined and set as follows\+: 
\begin{DoxyCode}
server.AssignPort(serverPort);
client.SetDestination(serverHost, serverPort);
\end{DoxyCode}
 where server\+Host could either be the hostname or the I\+P address of the server. U\+D\+P sockets update their destination to the origin of the last message received.

For the T\+C\+P case, the client has to additionally call the Connect() method, while the server is created using the \hyperlink{classosa_socket_server}{osa\+Socket\+Server} class. 
\begin{DoxyCode}
client.SetDestination(serverIP, serverPort);
client.Connect();
\textcolor{comment}{// or}
client.Connect(serverIP, serverPort);
\end{DoxyCode}
 The T\+C\+P server is defined using \hyperlink{classosa_socket_server}{osa\+Socket\+Server}, which calls an overloaded \hyperlink{classosa_socket}{osa\+Socket} constructor upon accepting a connection (see \hyperlink{classosa_socket_server}{osa\+Socket\+Server} class).

\begin{DoxyNote}{Note}
Please refer to os\+Abstraction\+Tutorial/sockets for usage examples. 

Disconnection is detected when a socket attempts to write to another socket and does not received an A\+C\+K.
\end{DoxyNote}
