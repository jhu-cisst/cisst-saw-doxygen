Define an Euler angle rotation for a space of dimension 3. 

There are several conventions for Euler angle rotations, which depend on the order of rotations about the axes, and whether the rotations are intrinsic (i.\+e., about the body\textquotesingle{}s coordinate frame) or extrinsic (i.\+e., about the world coordinate frame).

We use the common convention of three letters to define the order of (intrinsic) rotations. For example, Z\+Y\+Z refers to a rotation of $\phi$ (or $\alpha$) about Z, followed by a rotation of $\theta$ (or $\beta$) about Y\textquotesingle{}, followed by a rotation of $\psi$ (or $\gamma$) about Z\textquotesingle{}\textquotesingle{}. Here, the Y\textquotesingle{} and Z\textquotesingle{}\textquotesingle{} denote that the rotations are about the new (rotated) Y and Z axes, respectively. For convenience (and to conform to C++ naming rules), the \textquotesingle{} and \textquotesingle{}\textquotesingle{} are omitted from the naming convention. Note that when composing intrinsic rotations, the order is as follows\+: $R_Z(\alpha) R_Y(\beta) R_Z(\gamma)$. All angles are in radians.

The range of angles must still be determined. One possibility is to have $\phi$ and $\psi$ be in the range ( $-\pi$, $+\pi$\mbox{]}, and $\theta$ is in the range \mbox{[}0, $+\pi$\mbox{]}. For now, the Is\+Normalized method always returns true.

Because there are so many possible Euler angle conventions, we implement a base class, \hyperlink{classvct_euler_rotation3}{vct\+Euler\+Rotation3}, and then specialize it with derived classes that are templated by the \hyperlink{namespacevct_euler_rotation3_order_ac67e18ebf7cf8c53e5578514bff3efbb}{vct\+Euler\+Rotation3\+Order\+::\+Order\+Type} enum (see \hyperlink{vct_forward_declarations_8h}{vct\+Forward\+Declarations.\+h}).

Note that we could have also templated the Euler angle class by element type (double or float), but decided that this was not worthwhile -- all Euler angle rotation classes use double.
