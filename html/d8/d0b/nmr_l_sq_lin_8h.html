This function checks for valid input and calls the LAPACK function. The approach behind this defintion of the function is that the user creates a solution object from a code wherein it is safe to do memory allocation. This solution object is then passed on to this method along with the matrix whose LSqLin is to be computed. The solution object has members X and Work etc., which can be accessed through calls to method get*() along with adequate workspace for LAPACK. This function modifies the contents of matrix A. For details about nature of the solution matrices see text above.

Basic version of LSqLin where user provides the input matrix as well as storage for output and workspace needed by LAPACK. No mem allocation is done in this function, user allocates everything inlcuding workspace. The LSqLin method verifies that the size of the solution objects matchesc the input. See static methods in nmrLSqLinSolutionDynamic for helper functions that help determine the min workspace required. This function modifies the contents of matrix A.
